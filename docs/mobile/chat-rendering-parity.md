## Mobile chat rendering parity (Flutter)

- 2026-04-26: Ввод текста на мобильном: у `TextField`/`TextFormField` задано `textCapitalization` — **`sentences`** для сообщений, поиска, подписей, опросов и т.п.; **`words`** для ФИО/имени при регистрации и гостевого имени в митинге; **`none`** для email, паролей, телефона, URL ссылок, дат и логина (клавиатура не навязывает верхний регистр там, где он мешает).
- 2026-04-26: `ChatComposer` — строка ввода как поиск на списке чатов: высота 40, скругление 14, шрифт 16 / `FontWeight.w500`, `contentPadding` vertical 10, `cursorColor`/`cursorHeight` от темы, цвет текста и подсказки через `ColorScheme.onSurface` (светлая тема читаема). Модалка «Новая папка» (`chat_list_screen`): в светлой теме фон `surfaceContainerHigh`, те же `modalFg` для текста/границ; убран `Colors.white` у кнопки «Отмена» и у инициалов в аватаре без цвета.

### What is implemented
- **Форматирование текста** (паритет TipTap / `MessageEditor` + `FormattingToolbar.tsx`): в поле ввода хранится **HTML** с тегами как на вебе (`<strong>`, `<em>`, `<u>`, `<s>`, `<a href>`, `<span class="spoiler-text">`, `<code>`, `<blockquote>`). Перед отправкой — `sanitizeMessageHtml` (allowlist как `sanitize-message-html.ts`). Рендер в ленте — `messageHtmlToStyledSpans` (DOM `package:html`), спойлер по тапу, ссылки через `url_launcher`. Обычный текст без тегов при отправке оборачивается в `<p>…</p>` с `<br>` для переносов. Редактирование сообщения подставляет исходный HTML в `TextField`. Ветка обсуждений: тот же `prepareChatMessageHtmlForSend` при отправке (отдельной панели форматирования в `ThreadScreen` нет).
- **Стикеры и GIF** (паритет `ChatStickerGifPanel` / `UserStickersTab`): нижний лист по пункту «Стикеры и GIF» — вкладки **СТИКЕРЫ** / **GIF**; стикеры — переключатель **Мои** / **Общие** (`users/{uid}/stickerPacks/*` и `publicStickerPacks/*`), полоса паков, сетка 4 колонки (`CachedNetworkImage`), создание пака, долгое нажатие на пак — удаление, долгое нажатие на стикер в своём паке — удаление; **С устройства** / **В мой пак** — выбор изображений → лист выбора пака → загрузка в Storage `users/{uid}/sticker-packs/{packId}/…` + документ `items`. GIF — поиск через прокси веба `GET {TENOR_PROXY_BASE_URL}/api/tenor/search` (сборка: `--dart-define=TENOR_PROXY_BASE_URL=https://…`), тап — отправка `ChatAttachment` как на веб (`gif_{id}.gif`), закладка — сохранение URL в пак. Отправка в чат: `ChatRepository.sendTextMessage` с пустым текстом и одним вложением (`sticker_*` / `gif_*` по `userStickerItemToAttachment`, паритет `src/lib/user-sticker-packs.ts`). Репозиторий: `user_sticker_packs_repository.dart`, провайдер `userStickerPacksRepositoryProvider` в `app_providers.dart`. На мобильном **нет** квадратного кропа/HEIC-нормализации как в `user-sticker-packs-client.ts` (файлы грузятся как есть в пределах 8 МБ).
- **Message actions (long-press)**: паритет `MessageContextMenu.tsx` — `showMessageContextMenu` в `message_context_menu.dart`: полноэкранный blur-фон; **превью инициатора** **без** второго blur поверх него — для текста пузырёк, для сообщений с `attachments` тот же визуальный блок, что в ленте (`MessageAttachments`, ширина 272), чтобы меню примыкало к превью фото/стикера/видео и т.д., а не к заглушке «Вложение»; при тексте+медиа под превью — текстовый пузырёк. Далее стеклянная карточка ~272px: при `replyTo` сверху цитата, «ОТПРАВЛЕНО» / «ПРОЧИТАНО», горизонтальный скролл быстрых реакций (глифы **~28** logical, как изначально); превью сообщения «только эмодзи» над карточкой — тот же размер, что в ленте (`pureEmojiMessageFontSize` в `chat_emoji_only.dart` по `chatFontSize` → 64 / 76 / 86), действия по высоте контента. Реакции: `ChatRepository.toggleMessageReaction`; копирование: `Clipboard` + тост. `message_action_sheet.dart` реэкспортирует модуль.
- **Bulk selection**: после «Выбрать» или из режима выбора — тап по пузырьку переключает выделение; верхняя панель — переслать выбранные или массовое удаление (только свои неудалённые). `chat_selection_app_bar.dart`.
- **Forward**: `ChatForwardScreen` (`/chats/forward`, `extra: List<ChatMessage>`) — предпросмотр текста, поиск; получатели: **группы** из индекса и **личные чаты только с контактами** (`userContacts.contactIds`), строка «в контактах без чата» открывает DM через `createOrOpenDirectChat`; чат «Избранное» (один участник — вы) **исключён**; имена/аватары из `users/*` и `Conversation.participantInfo` / `photoUrl` для групп.
- **Pins**: до 20 закреплённых, список как на веб (`pinnedMessages` + legacy `pinnedMessage` в модели); полоса превью `chat_pinned_strip.dart`, логика списка — `pinned_messages_helper.dart` (`buildChatPinSyncFlatRows`, `pickPinnedBarIndexForViewport` — паритет `src/lib/chat-pinned-messages.ts`). Какой закреп показан в полосе, синхронизируется со скроллом (оценка видимого диапазона по `ScrollController` + плоские индексы «дата/сообщение»); тап по превью ведёт к **текущему** показанному закрепу, затем индекс циклически сдвигается как на вебе `(i - 1 + n) % n` с паузой синхронизации ~900 ms. Тап по тексту полосы (не по «×») — `ChatScreen._scrollToMessageId` + `GlobalKey` на строке в `ChatMessageList`.
- **Edited / forwarded UI**: у пузырька показывается «изм.» при непустом `updatedAt`; пересланные — строка «Переслано от …» по `forwardedFrom`.
- **Reply flow (send)**: long-press a message → «Ответить» → preview bar above composer → new message is written with `replyTo` in Firestore (same shape as web).
- **Edit own text**: long-press → «Изменить» → полоса `ComposerEditingBanner` + тот же `TextEditingController`, что и для отправки; «Отправить» вызывает `updateMessageText` (без отдельного контроллера в `AlertDialog` — избегает dispose/listener assert при закрытии диалога).
- **Message order / scrolling**: сортировка по `createdAt` + `id`, список — `CustomScrollView` + группы по дням: у каждого дня `SliverPersistentHeader` (pinned) с капсулой даты; вниз в начале — несколько кадров `jumpTo(maxScrollExtent)` пока экстент стабилен (меньше скачка после декода картинок); подгрузка старых: только по **жесту пользователя** / `ScrollEnd` у верхнего края (не по программному `ensureVisible`) + короткий cooldown в `ChatScreen` после восстановления якоря; при **росте limit без новых сообщений** — без `ensureVisible`, позиция не сбрасывается; ветка `loading` у `messagesProvider` при непустом `_sortedAscCache` продолжает показывать список (не спиннер на весь экран), чтобы смена `limit` не dispose’ила `ChatMessageList` и не вызывала повторный jump вниз.
- **Ответ (reply)**: `MessageReplyPreview` без отступа снизу (с пузырём слитно); `replyPreviewToBodyGap` только **сверху** рамки, если нет «Переслано…» (иначе зазор даёт padding под пересланным).
- **Chat polls**: сообщения с `chatPollId` — `MessageChatPoll` + `ChatGlassPanel`; меню «⋯» у **модераторов** — `chatDarkPopupMenuScope` + скруглённый полупрозрачный `PopupMenuTheme` (`chatGlassPopupMenuShape`). У активного опроса кнопка **«Переголосовать»** на карточке для всех, кто уже голосовал (снятие голоса через Firestore). Заглушка веба «📊 Опрос» / «Опрос» (`isChatPollStubCaptionPlain`) не даёт отдельный синий пузырёк; время и галочки при `showTimestamps` встроены в правый нижний край стеклянной карточки (`embedMessageStatus`). Реальный подпись к опросу — по-прежнему в `textBubble` под карточкой. «Опрос + медиа» без видимого текста: у вложений `showTimestamps: false`, метка на карточке опроса.
- **Location share**: поле `locationShare` в `ChatMessage` — `MessageLocationCard` (превью static map или заглушка, таймер для live, тап → `SharedLocationMapScreen`). При отправке с мобильного `buildChatLocationShareFromPosition` задаёт `staticMapUrl` через `buildChatLocationStaticPreviewUrl`: при `--dart-define=GOOGLE_MAPS_API_KEY=...` — Google Static Maps (как веб с `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY`), иначе URL одного тайла `tile.openstreetmap.org` (надёжнее, чем `staticmap.openstreetmap.de` с дефолтным Dart User-Agent). Загрузка превью в `ChatCachedNetworkImage` с заголовком `User-Agent: LighChatMobile/1.0 …` (требование политики OSM). Текст «трансляция завершена» — `ChatGlassPanel` + белый шрифт. Полноэкранная карта: `loadHtmlString` + Leaflet/OSM (`buildOpenStreetMapLeafletHtml` в `google_maps_urls.dart`), `User-Agent: LighChatMobile/1.0` — Google embed в WKWebView часто даёт чёрный экран. Превью: прозрачный `Material`; сообщение **только с геолокацией** без пустого исходящего пузыря. Порядок: переслано → ответ → карта → остальное. `LiveLocationStopBanner` над композером. **Отправка из композера**: скрепка → Локация → `share_location_sheet.dart` (режим «Как делиться»; без иконки «×» в заголовке — закрытие только «Отмена») → `Geolocator` → подтверждение `location_send_preview_sheet.dart` (мини-карта OSM в WebView, координаты, точность) → `ChatRepository.sendLocationShareMessage`. Если на iOS `MissingPluginException` для `geolocator_apple`, выполните `cd mobile/app/ios && pod install` и пересоберите (в `Podfile.lock` должен появиться pod `geolocator_apple`). **Обновление live на устройстве**: `LiveLocationFirestoreSync` в `main.dart` builder — стрим позиции с троттлингом ~15 с в Firestore (паритет `LiveLocationProvider.tsx`).
- **Несколько фото в одном сообщении**: мозаика в `message_attachments.dart` + `chat_attachment_mosaic_layout.dart`; блок фото и блок видео/кружков разнесены `RepaintBoundary` + верхний `Padding`, чтобы при оверлеях (контекст-меню и т.п.) слои вёрстки не «слипались». — `Row`/`Column` без `GridView`+`shrinkWrap`; высоты рядов от **aspect ratio** (`ChatAttachment.width`/`height`, иначе 1:1): 2 фото — одна высота, ширины ∝ w/h; 3 — полная ширина + ряд из двух; 4 — 2×2; 5–9 — 3+2 … 3×3; до **9** превью, сверху на последней плитке «+N». Два фото в исходящих прижаты вправо (`Align`). Сетевые картинки вложений — `ChatCachedNetworkImage` (`cached_network_image`: диск + память), чтобы при обратном скролле не тянуть файл снова с сети; то же для превью ответа, аватаров (`ChatAvatar`, список чатов), статической карты в `MessageLocationCard`, HTTP-обоев в `ChatScreen`. В `main.dart` подняты лимиты `PaintingBinding.instance.imageCache` для декодированных растров.
- **Video inline preview**: первый кадр через `video_player`; при достаточной видимости в ленте — автозапуск/пауза по порогам `visibility_detector`. Поверх плеера — `GestureDetector` с `HitTestBehavior.translucent` и `IgnorePointer` на `VideoPlayer`, чтобы вертикальные свайпы доходили до `CustomScrollView` (без «залипания» скролла на видео). `AspectRatio` в ленте берётся из `ChatAttachment.width`/`height`, если они есть, а не с контроллера после init — меньше скачка высоты рядом с кружком. WebM на iOS по-прежнему без превью — открытие в VLC viewer.
- **Полноэкранная галерея медиа** (паритет `media-viewer.tsx`): тап по фото/видео из сетки (не стикер/GIF/кружок) открывает `ChatMediaViewerScreen` — размытый фон по текущему кадру (для видео — градиент), шапка (назад, имя, «n / N», дата по-русски), меню ⋮ и нижние кнопки: Ответить, Переслать, Удалить (только свои); **Отправить (во внешние приложения)** и **Сохранить** — скачивание во временный файл и системный share sheet (`share_plus`). Горизонтальная прокрутка между всеми подходящими вложениями в загруженной истории чата (`collectChatMediaGalleryItems` по `_sortedAscCache`, дедуп по `url`). Изображения — `InteractiveViewer` (pinch-zoom; при зуме горизонтальный свайп галереи отключён). Видео в галерее: `video_player` + прогресс; WebM на iOS — встроенный VLC как в превью. В `ThreadScreen` та же галерея по объединённому списку родитель + ответы; кнопка «Ответить» в просмотрщике скрыта (нет превью ответа в композере ветки).
- **Видеокружки** (имя вложения `video-circle_*`, как на вебе): запись — полноэкранный `video_circle_capture_page.dart` (круг 192 logical, зеркальный превью фронтальной камеры, стоп; затем превью записи — корзина / отправка). Отправка: `video-circle_{timestamp}.mp4|.mov` → Storage → `sendTextMessage` без текста. В ленте `message_video_circle_player.dart`: превью кадра (лёгкий seek по метаданным); при **воспроизведении** — полноэкранный `Overlay` с затемнением, круг **по центру экрана**, диаметр **min(2×192, ширина/высота экрана с отступами)**; в ленте остаётся заглушка 192×192; длительность и mute только после касания/наведения (~2.8 с), **внутри** `ClipOval` (не «уезжают» за пределы квадрата круга); кольцо прогресса при play или показе контролов. `visibility_detector` на заглушке — пауза, если видимая доля ниже ~0.12; параллельно играет только один кружок (`ValueNotifier<String?>` слотов `{messageId}_vc_{i}` в `ChatMessageList`). Свернуто: бейдж снизу — длительность + статус доставки. В контекст-меню для кружка создаётся локальный нотификатор внутри `MessageAttachments` (без списка чата). **Веб (скролл рядом с сеткой + кружком):** в `ChatMessageItem.tsx` полноэкранная центровка всей строки при play включается только если в сообщении нет сетки `MessageMedia` (`useFullBleedVideoCirclePlaying`); плюс в `ChatWindow`/`ThreadWindow` при росте резерва под кружок не вызывается принудительный `scrollToIndex` в конец, а в `VideoCirclePlayer` после `wheel`/`touchmove` на ленте кратко подавляется `scrollIntoView`. **Мобилка:** в `message_video_circle_player.dart` при воспроизведении в ленте не вызывается `setState` на каждом кадре — только `markNeedsBuild` оверлея.
- **Voice waveform**: при узком `Expanded` волна масштабируется (`FittedBox.scaleDown`), без overflow. **VLC на iOS Simulator** отключён (`chatMediaNeedsVlcOnIos` → `just_audio`/плейсхолдер), чтобы не сыпался `channel-error`.
- **Date separators**: одна плавающая капсула (`ChatBlurredDateCapsule`) поверх ленты; подпись — календарный день, который «активен» у верхнего края viewport (маркер `GlobalKey` на первом сообщении каждого дня + `RenderAbstractViewport` / `localToGlobal` относительно viewport). Раньше у каждого дня был свой `SliverPersistentHeader` — убрано, чтобы не копить несколько тегов при скролле.
- **Reply preview**: renders `replyTo.senderName`, preview text, optional `mediaPreviewUrl`; тап по рамке — `_scrollToMessageId`: id в `_sortedAscCache`, грубый `jumpTo` по индексу, `jumpScrollBoostMessageId` и увеличенный `cacheExtent` (высокие сообщения вне viewport), затем `Scrollable.ensureVisible`.
- **Outgoing text + time** (как Telegram): `Stack` + `Padding(right: reserve)` под текст (слева, с переносами) + `Positioned(right:0,bottom:0)` для времени и галочек; `ConstrainedBox(maxWidth)` + `IntrinsicWidth` для узкого пузыря на коротких сообщениях. Резерв справа считается через `TextPainter` (`_outgoingBubbleFooterReserveWidth`), чтобы «изм.» + время + галочки не наезжали на текст при правке. При **ответе** (`replyTo`) колонка сообщения тоже оборачивается в `IntrinsicWidth`: `MessageReplyPreview` без `Row`+`Expanded`, с `mainAxisSize: min` и ограничением текста по `messageBubbleMaxWidth`, чтобы рамка цитаты сжималась вместе с пузырём. Если есть «опрос + текст» / «медиа + текст», родительская `Column` не `stretch` на всю ширину (`CrossAxisAlignment.end` для исходящих), иначе пузырёк растягивается до `maxWidth` и время оказывается у текста, а не у правого края.
- **Pure emoji (без пузыря)**: только эмодзи — `Align` вправо для своих, влево для входящих (родительская колонка `stretch` иначе тянет текст на всю ширину); размер шрифта — `pureEmojiMessageFontSize(fontSize)` (то же над контекст-меню).
- **Deleted messages**: renders “Сообщение удалено” (no text/attachments/reactions).
- **Reactions**: renders chips from `reactions` map (emoji + count), highlights when the current user reacted.
- **Partner profile**: тап по аватару/заголовку в шапке → bottom sheet как на веб `ChatParticipantProfile` (упрощённо): строка статуса / `~username`, сворачиваемый блок «Контакты и данные» (email, телефон, день рождения, о себе, роль — с учётом `privacySettings` как на веб), «Добавить в контакты» (`userContacts` + `arrayUnion`), для групп — участники / редактирование (пока «скоро»), меню медиа/избранное/обсуждения/уведомления/тема/приватность/шифрование (большинство — «скоро»; счётчик «Медиа» из последних 100 сообщений), «Создать группу с…», «Покинуть группу» (скоро). Кнопка «Поделиться» копирует `conversationId` в буфер. Данные собеседника: расширенный `UserProfile` из `users/*`; шапка чата подтягивает онлайн/`lastSeen` и корректные заголовки для «Избранное».

### Data model notes
Flutter model `ChatMessage` is extended to parse:
- `replyTo` (ReplyContext)
- `isDeleted`
- `reactions` (legacy `string[]` or `{ userId, timestamp }[]`)
- `chatPollId` (ссылка на документ опроса в подколлекции `polls`)
- `locationShare` (`ChatLocationShare`: lat/lng, mapsUrl, staticMapUrl, liveSession.expiresAt, …)
- **Threads (обсуждения)**: `threadCount`, `unreadThreadCounts`, `lastThreadMessageText`, `lastThreadMessageSenderId`, `lastThreadMessageTimestamp` — паритет с веб `ChatMessage`; на чате `Conversation.unreadThreadCounts` для бейджа в шапке.

See: `mobile/packages/lighchat_models/lib/lighchat_models.dart` (в т.ч. опциональные поля `Conversation`: `description`, `adminIds`, `createdByUserId`, `e2eeEnabled`, `e2eeKeyEpoch` для сводки шифрования в профиле).

### Threads / обсуждения (web parity)
- Подколлекция: `conversations/{id}/messages/{parentId}/thread/{threadMsgId}`; отправка и обновление родителя/беседы — `ChatRepository.sendThreadTextMessage`, стрим — `watchThreadMessages` (`mobile/packages/lighchat_firebase/lib/src/chat_repository.dart`).
- UI: `thread_screen.dart` (корневое сообщение + ответы + композер), `conversation_threads_screen.dart` (список веток с `threadCount > 0`). Фон ветки — `ChatWallpaperBackground` с тем же правилом, что у `ChatScreen`: непустое `chatConversationPrefs.chatWallpaper`, иначе `chatSettings.chatWallpaper` (`effective_chat_wallpaper.dart` + `threadWallpaperBackdrop` в `thread_screen.dart`).
- Навигация: `/chats/:conversationId/threads`, `/chats/:conversationId/thread/:parentMessageId` (опционально `extra: ChatMessage` для родителя); меню сообщения «Обсудить», бейдж ответов под пузырьком (`ChatMessageList.onOpenThread`), иконка в `ChatHeader`, строка «Обсуждения» в `chat_partner_profile_sheet.dart`.
- Сообщения в ветке пишут `createdAt` как **ISO-строку** (как веб `ThreadWindow`), чтобы `orderBy('createdAt')` оставался согласованным с данными из браузера.

### Timestamp policy (IMPORTANT)
Mobile currently writes `createdAt` using **device time** (`Timestamp.now()` on the client) to match “time of sender device”.
This can drift if a device clock is wrong; if you later need canonical time, switch back to `serverTimestamp` and render a separate `clientCreatedAt`.

### How to test quickly
1. Open any chat with existing `replyTo`, `reactions`, `isDeleted` data.
2. Verify:
   - deleted messages show stub only
   - reply preview shows sender name + text
   - reactions chips show correct emoji counts
   - newest messages appear near the composer
3. Long-press: **изменить** — текст в основном поле ввода + полоса «РЕДАКТИРОВАНИЕ СООБЩЕНИЯ» (не модальное окно); удаление своего, закрепление, пересылка, вход в режим выбора.
4. В режиме выбора: переслать несколько сообщений; удалить только если все выбранные — ваши и не удалённые.
5. Закрепление: полоса сверху, снятие крестиком; не более 20 закреплённых.
