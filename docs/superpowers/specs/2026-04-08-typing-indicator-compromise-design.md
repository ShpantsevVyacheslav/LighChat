# Компромиссное восстановление индикатора "Печатает..."

## Контекст и цель

После оптимизаций из `b2b83422` индикатор "Печатает..." в списке чатов почти исчез: подписка на `conversations/{id}/typing` включается только для выбранного чата (`isSelected`).

Цель: вернуть индикатор для списка чатов без возврата к дорогому режиму "подписка на все диалоги всегда".

## Принятый подход

Выбран компромиссный вариант **visibility-gated typing**:

- Подписка на typing включается для карточки чата, если:
  - чат выбран (`isSelected`), или
  - карточка видна в viewport списка и вкладка браузера активна.
- Для невидимых карточек и фоновой вкладки listener отключается.

Это возвращает UX индикатора в живом списке и ограничивает количество активных realtime-listener'ов.

## Почему не другие варианты

- Platform-gated (разное поведение для iOS PWA и остальных платформ) отклонен: усложняет поддержку и делает UX непредсказуемым между устройствами.
- Soft sampling (редкий опрос вместо realtime) отклонен: индикатор запаздывает, что заметно ухудшает ощущение "живого" чата.

## Точки изменений

### `src/components/chat/ConversationItem.tsx`

- Добавить состояние видимости элемента `isInViewport` через `IntersectionObserver`.
- Добавить состояние активности вкладки `isPageVisible` через `document.visibilityState` и событие `visibilitychange`.
- Вычислять `typingEnabled` как:

```ts
const typingEnabled = isSelected || (isInViewport && isPageVisible);
```

- Передавать `typingEnabled` в `useConversationTypingOthers(...)` вместо `isSelected`.
- Визуальная часть текста "Печатает..." не меняется.

### `src/hooks/use-conversation-typing-others.ts`

- Изменений в логике чтения Firestore не требуется.
- Текущий контракт с `enabled` уже соответствует задаче.

## Data flow

1. Карточка чата рендерится в списке.
2. Вычисляются `isInViewport` и `isPageVisible`.
3. Формируется `typingEnabled = isSelected || (isInViewport && isPageVisible)`.
4. При `typingEnabled=true` хук включает `onSnapshot` на `conversations/{conversationId}/typing`.
5. При `typingEnabled=false` хук останавливает подписку и сбрасывает локальный флаг typing.

## Ошибки и устойчивость

- Ошибки listener'а продолжают обрабатываться текущим fallback (`setOthersTyping(false)`).
- Доступ к браузерным API (`document`, `IntersectionObserver`) выполняется только на клиенте внутри `useEffect`.
- На платформах без `IntersectionObserver` применяется безопасный fallback: считать элемент видимым, чтобы не ломать функциональность typing.

## Проверка результата

Ручной сценарий:

1. Открыть список чатов на активной вкладке: у видимых карточек снова появляется "Печатает...".
2. Прокрутить список: ушедшие из viewport карточки перестают обновлять typing.
3. Переключить вкладку в background: typing-обновления останавливаются.
4. Вернуться во вкладку: typing-обновления восстанавливаются.
5. Проверить выбранный чат: его typing работает независимо от viewport.

## Ожидаемый эффект

- Восстановление ключевого UX-сигнала "кто печатает" в списке чатов.
- Снижение realtime-нагрузки относительно режима "слушать все".
- Отсутствие изменений в схеме данных, типах и правилах Firestore.
